# Kasteran* ‚ÄĒ Contribution FAQ
¬© Lois-Kleinner & 0-1.gg 2026

## How can I contribute?

There are many ways to contribute:
- **Code**: Compiler, stdlib, runtime, tooling
- **Documentation**: API docs, tutorials, guides
- **Translation**: Localize documentation and tooling
- **Bug reports**: Submit detailed bug reports
- **Feature requests**: Suggest improvements
- **Community support**: Help others on forums and chat
- **Testing**: Run test suites, report issues
- **Reviews**: Review pull requests and RFCs

## What is the PR process?

1. Find or create an issue
2. Fork the repository
3. Create a feature branch
4. Make changes with tests
5. Run tests locally
6. Submit a pull request
7. Respond to review feedback
8. PR is merged

## What are the coding standards?

- Follow existing code style
- 4-space indentation
- 100 character line limit
- Meaningful variable names
- Document public APIs
- Write tests for new code
- Conventional commits for commit messages

## What testing is required?

- Unit tests for all new functionality
- Integration tests for features
- At least 80% code coverage
- All tests must pass before merge
- Linting (clippy) must pass
- Formatting (fmt) must pass

## Do I need to be an expert?

No! Contributions from all skill levels are welcome:
- **Good first issues**: Labeled for beginners
- **Mentorship**: Experienced contributors help newcomers
- **Documentation**: Great starting point for beginners
- **Bug fixes**: Often easier than new features

## Can I contribute to documentation?

Absolutely! Documentation contributions are highly valued:
- Fix typos and errors
- Improve clarity
- Add examples
- Translate to other languages
- Write tutorials and guides

## How are contributions recognized?

- Contributors listed in README
- Release notes mention significant contributions
- Outstanding contributors invited to core team
- Swag for significant contributions

## Conclusion

Everyone can contribute to Kasteran*. Start with a small issue or documentation fix, and join the community of contributors building the language.
