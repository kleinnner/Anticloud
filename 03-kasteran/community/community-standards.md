# Kasteran* ‚ÄĒ Community Standards
¬© Lois-Kleinner & 0-1.gg 2026

## Overview

Community standards establish the norms and expectations for how community members interact. Kasteran* community standards foster a productive, respectful, and inclusive environment where everyone can contribute effectively.

## Communication Norms

### Be Respectful
- Treat others with respect and dignity
- Assume good faith in others' intentions
- Disagree constructively, not personally
- Avoid aggressive or confrontational language

### Be Inclusive
- Use inclusive language (avoid gendered terms)
- Consider different cultural perspectives
- Welcome newcomers and beginners
- Accommodate different communication styles

### Be Clear
- Write clearly and concisely
- Provide context for your comments
- Use appropriate channels for topics
- Keep discussions on topic

### Be Patient
- Not everyone has the same experience level
- Language barriers may exist
- Time zones affect response times
- Contributors have varying availability

## Communication Channels

| Channel | Purpose | Expectations |
|---|---|---|
| GitHub Issues | Bug reports, feature requests | Technical, focused |
| GitHub Discussions | Questions, ideas, general | Welcoming, helpful |
| Discord | Real-time chat, quick questions | Friendly, informal |
| Forum | Long-form discussions | Thoughtful, detailed |
| Social media | Announcements, community | Professional, positive |

## Review Expectations

### For Reviewers
- Provide constructive feedback
- Focus on code, not the person
- Explain the reasoning behind suggestions
- Be timely in reviews (within 3 days)
- Acknowledge good contributions

### For Contributors
- Be open to feedback
- Explain your design decisions
- Address all review comments
- Ask for clarification if needed
- Be patient during the review process

### Review Guidelines
```
review:
  do:
    - Comment on design and logic
    - Suggest improvements
    - Approve when satisfied
  don't:
    - Insult or belittle
    - Demand changes without explanation
    - Block without clear reasoning
    - Make personal comments
```

## Recognition

### Contribution Recognition
We recognize contributions through:
- **Contributors list**: All contributors listed in README
- **Release notes**: Significant contributions mentioned
- **Bug fixes**: Fixed-by attribution in commits
- **Core team**: Outstanding contributors invited to core team

### No-Toxicity Policy
We do not tolerate:
- Harassment or bullying
- Discrimination or prejudice
- Aggressive or threatening behavior
- Gatekeeping or elitism
- Dismissive attitudes toward questions

## Mentorship

### Mentorship Program
Experienced contributors mentor newcomers:
- Pair programming sessions
- Code review mentoring
- RFC writing guidance
- Project onboarding support

### Expectations
Mentors provide:
- Patient, constructive guidance
- Regular check-ins
- Encouragement and support
- Growth opportunities

## Conflict Resolution

### Process
1. Direct communication between parties
2. Moderation team mediation
3. Code of Conduct enforcement
4. Escalation to core team

### Principles
- Address issues early
- Focus on behavior, not people
- Seek understanding first
- Find mutually acceptable solutions

## Conclusion

Kasteran* community standards create a productive, respectful environment. By following these norms, we build a community where everyone can contribute, learn, and grow together.
