# Kasteran* — Governance Model
© Lois-Kleinner & 0-1.gg 2026

## Overview

The Kasteran* governance model defines how decisions are made, who has authority, and how the project is managed. Our governance is designed to be transparent, meritocratic, and community-driven while ensuring long-term project sustainability.

## Governance Structure

```
┌────────────────────────────────────────────────┐
│                Community                       │
│  (Contributors, users, stakeholders)           │
└──────────────────┬─────────────────────────────┘
                   │
┌──────────────────┴─────────────────────────────┐
│             Core Team                          │
│  (Strategic decisions, roadmap, governance)    │
└──────────────────┬─────────────────────────────┘
                   │
┌──────────────────┴─────────────────────────────┐
│            Maintainers                        │
│  (Technical decisions, code review, releases)  │
└──────────────────┬─────────────────────────────┘
                   │
┌──────────────────┴─────────────────────────────┐
│           Contributors                        │
│  (Code, docs, issues, community support)       │
└────────────────────────────────────────────────┘
```

## Contributors

Anyone can become a contributor by:
- Submitting PRs with code or documentation
- Reporting and triaging issues
- Helping others in community channels
- Translating documentation
- Organizing community events

### Rights
- Submit pull requests and issues
- Participate in discussions
- Vote on RFCs (after 3 months of contribution)

### Path to Maintainer
- Consistent, high-quality contributions
- Understanding of project architecture
- Positive community reputation
- Core team nomination and vote

## Maintainers

Maintainers are responsible for day-to-day technical management.

### Responsibilities
- Review and merge pull requests
- Triage and resolve issues
- Manage releases
- Mentor new contributors
- Participate in technical discussions

### Appointment
- Nominated by existing maintainers or core team
- Vote by core team (simple majority)
- Term: No fixed term (ongoing evaluation)

### Removal
- Inactivity for 6+ months
- Code of Conduct violations
- Vote by core team (2/3 majority)

## Core Team

The core team provides strategic direction and governance.

### Responsibilities
- Set project roadmap and vision
- Approve major design decisions
- Manage project resources
- Resolve conflicts
- Appoint and remove maintainers
- Manage Code of Conduct enforcement

### Composition
- Founding members (Lois-Kleinner & 0-1.gg)
- Long-term maintainers (invited)
- Community representatives (elected)

### Appointment
- Invitation by existing core team members
- Vote by core team (2/3 majority)
- Term: 2 years, renewable

### Removal
- Inactivity for 12+ months
- Code of Conduct violations
- Vote by core team (unanimous minus the member)

## Decision-Making

### Lazy Consensus
Default decision-making model:
- Proposal is made (via RFC, issue, or discussion)
- Discussion period (minimum 2 weeks for significant changes)
- If no objections, consensus is assumed
- Objections are addressed through discussion

### Voting
When consensus cannot be reached:
- Simple majority for most decisions
- 2/3 majority for constitutional changes
- Unanimous for removal decisions

### RFC Process
Significant changes follow the RFC process:
- Published for community review
- Minimum 2-week discussion period
- Final Comment Period (1 week)
- Core team decision

## Conflict Resolution

### Escalation Path
1. Direct discussion between parties
2. Maintainer mediation
3. Core team intervention
4. Code of Conduct enforcement

### Appeals
Decisions can be appealed to the core team.

## Transparency

All governance activities are transparent:
- Meeting notes published
- Decisions documented
- Rationale provided
- Dissenting opinions recorded

## Conclusion

The Kasteran* governance model balances community participation with project stability. The tiered structure provides clear paths for contribution, leadership, and decision-making while ensuring accountability and transparency.
