## How to Ask Good Questions

To get helpful answers quickly, follow this template when asking questions:

### Question Template

```
**Question:** [Brief, specific question in 10 words or less]

**Context:**
- Kamelot version: [output of `kamelot --version`]
- OS: [OS name and version]
- Hardware: [CPU, RAM, GPU]
- Config: [relevant parts of config.toml, sanitized]

**What I'm trying to do:**
[1-2 sentences describing your goal]

**What I've tried:**
[Steps you've already taken]

**What happens:**
[Error messages, unexpected behavior, logs]

**What I expect to happen:**
[Desired outcome]

**Logs (relevant section):**
```
[Paste relevant logs here]
```
```

### Good vs Bad Questions

**Bad:** "Kamelot doesn't work. Help!"

**Good:** "Kamelot v0.1.0 on Ubuntu 24.04 — Qdrant connection refused at localhost:6333 after Docker restart. I've checked that the container is running (`docker ps` shows qdrant), but `curl localhost:6333/health` returns connection refused. My config has `qdrant.url = "http://localhost:6333"`. What should I check next?"

### Before Asking

1. **Read the documentation**: Most common questions are answered in the docs
2. **Search existing resources**: GitHub Discussions, Discord history, wiki
3. **Run `kml doctor`**: The diagnostic tool catches common issues
4. **Check logs**: `kml logs --tail 100` often contains the answer
5. **Try mock mode**: `kml start --model mock` isolates model-related issues

## Community Guidelines

### Be Respectful

Treat all community members with respect. Disagreement is fine; personal attacks are not. Harassment, discrimination, and hate speech are not tolerated.

### Stay On Topic

Keep discussions relevant to the channel or thread. Off-topic conversations can be moved to `#off-topic`.

### Don't Spam

Do not post the same question in multiple channels. Do not advertise products or services unrelated to Kamelot.

### No Piracy

Do not ask for or share copyrighted material, including paid software.

### Respect Privacy

Do not share personal information about yourself or others. Do not post logs or configs containing unredacted secrets.

### Follow the Code of Conduct

All community participation is governed by the [Code of Conduct](#code-of-conduct).

### Enforcement

Violations of community guidelines may result in:
1. Warning (first offense)
2. Temporary mute (second offense)
3. Temporary ban (third offense)
4. Permanent ban (severe or repeated violations)

## New Contributor Onboarding

### Step 1: Pick an Area

Kamelot has several areas where contributions are welcome:

| Area | Skills Needed | Good First Issue |
|------|--------------|------------------|
| Documentation | Writing, technical communication | FAQ expansion, typos |
| Bug fixes | Rust, debugging | Labeled "good first issue" |

| UI/UX | GPU programming, design | UI widget improvements |
| Testing | Software testing | Test coverage gaps |
| Translations | Language skills | Documentation translation |
| Community support | Communication, patience | Answering questions |

### Step 2: Set Up Development Environment

```bash
# Clone the repository
git clone https://github.com/kamelot/kamelot.git
cd kamelot

# Build
cargo build

# Run tests
cargo test

# Run with mock backend
cargo run -- start --model mock
```

See `CONTRIBUTING.md` for detailed setup instructions.

### Step 3: Find an Issue

- Browse [GitHub Issues](https://github.com/kamelot/kamelot/issues)
- Filter by label: `good first issue`, `help wanted`, `documentation`
- Comment to express interest and ask questions
- An existing contributor will help you get started

### Step 4: Submit Your Contribution

1. Fork the repository
2. Create a branch for your changes
3. Make your changes
4. Run tests: `cargo test`
5. Submit a Pull Request
6. Respond to review feedback
7. Celebrate when merged!

## Community Health and Safety

### Reporting a Concern

If you encounter behavior that violates the Code of Conduct:

1. **Document**: Take screenshots or save relevant messages
2. **Report**: Email conduct@kamelot.ai or DM a moderator on Discord
3. **Include**: What happened, when, who was involved, and any evidence
4. **Expect**: Acknowledgment within 24 hours, resolution within 72 hours

### What Happens After a Report

1. Report is reviewed by the moderation team
2. Both parties are given an opportunity to provide their perspective
3. Moderation team determines if a violation occurred
4. Appropriate action is taken (warning, mute, ban)
5. Reporter is notified of the outcome (privacy permitting)

### Confidentiality

All reports are handled confidentially:
- Reporters remain anonymous to the broader community
- Only the moderation team knows the reporter's identity
- Details of moderation actions are not shared publicly
- Retaliation against reporters is a serious violation

Your GitHub username changes automatically across all repositories. If you change your GitHub username:
1. Notify in `#general` on Discord

### I'm Being Harassed in the Community. What Should I Do?

1. Do not engage with the harasser
2. Screenshot the messages
3. Report to conduct@kamelot.ai or DM a moderator
4. Moderators will handle the situation
5. If urgent, mention @Moderator in the channel

### Can I Promote My Product in the Community?

The Kamelot community is for discussion about Kamelot and related technologies. Product promotion is allowed only if:
- It directly relates to Kamelot (integration, plugin, use case)
- It's posted in the appropriate channel (#showcase or #off-topic)
- It's not spam (repeated posting, same content)
- You're an active community member, not just here to promote

Unsolicited direct messages promoting products are not allowed.

### How Do I Delete My Community Account?

Community accounts are separate from Kamelot software:
- **Discord**: Delete your Discord account via Discord settings
- **GitHub**: Delete your GitHub account via GitHub settings
- **Wiki**: Contact wiki@kamelot.ai for account deletion
- **Mailing list**: Unsubscribe link at the bottom of every email

Deleting your community accounts does not affect your Kamelot software installation.
