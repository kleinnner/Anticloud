# Grant Writing Guide

## Overview

Best practices for writing grant proposals for API-OSS.

## Structure

```markdown
1. Executive Summary (1 page)
   - Problem, solution, impact
   - Requested amount

2. Technical Approach (3-5 pages)
   - Architecture
   - Innovation
   - Implementation plan

3. Impact (2-3 pages)
   - Market need
   - Economic impact
   - Societal benefit

4. Team (1-2 pages)
   - Qualifications
   - Track record

5. Budget (1-2 pages)
   - Personnel
   - Equipment
   - Travel
   - Overhead

6. Timeline (1 page)
   - Milestones
   - Deliverables
```

## Tips

```
- Read the call document carefully
- Address all evaluation criteria
- Use clear, jargon-free language
- Quantify impact where possible
- Include risk mitigation
- Get early feedback
- Follow formatting requirements exactly
```

## Common Mistakes

```
- Generic proposals (not tailored)
- Overpromising
- Weak dissemination plan
- Missing budget justification
- Ignoring evaluation criteria
- Poor formatting
- Late submission
```
