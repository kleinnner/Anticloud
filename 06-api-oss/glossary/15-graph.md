# Glossary 15: Graph Glossary

## Terms

### Graph Engine
- Knowledge graph visualization system
- Represents entities (nodes) and their relationships (edges)

### Node
- An entity or concept in the knowledge graph
- Examples: person, document, organization, project, topic

### Edge
- A relationship between two nodes
- Examples: "works on", "mentions", "belongs to", "depends on"

### Node Type
- Category of node (Person, Document, Organization, etc.)
- Defines icon, color, and available properties

### Edge Type
- Category of relationship (WorksOn, Mentions, DependsOn, etc.)
- Defines directionality and label

### Graph Layout
- Algorithm that determines node positioning
- API-OSS supports: force-directed, hierarchical, circular, grid, radial

### Force-Directed Layout
- Physics-based layout: nodes repel, edges attract
- Default layout for knowledge graphs

### Graph Filtering
- Showing/hiding nodes based on criteria
- Filter by: type, properties, time range, search query

### Graph Search
- Finding nodes and edges within the graph
- Supports: text search, property filter, path finding

### Graph Export
- Exporting graph as image or data
- Formats: PNG, SVG, JSON, GraphML, CSV

### Graph History
- Chronological view of graph changes
- Track how knowledge has evolved over time

### Subgraph
- A subset of nodes and edges from the full graph
- Created by graph filtering or traversal

### Graph Traversal
- Walking through connected nodes
- Used for path finding and influence analysis

### Graph Statistics
- Metrics about the graph
- Node count, edge count, density, central nodes, clusters

### Graph Annotation
- Adding notes and metadata to nodes
- Collaborative graph enrichment
