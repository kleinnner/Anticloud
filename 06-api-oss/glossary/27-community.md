# Glossary 27: Community Glossary

## Terms

### Contributor
- Anyone who contributes to API-OSS
- Types: code, docs, design, community support, translation

### Maintainer
- Trusted contributor with commit access
- Reviews PRs, triages issues, guides contributors

### Core Team
- Small group driving project direction
- Makes final decisions on RFCs and roadmap

### Community Manager
- Person responsible for community health
- Organizes events, moderates, supports contributors

### RFC (Request for Comments)
- Structured proposal for significant changes
- Community discussion period before decision

### Governance
- Rules and processes for project decisions
- Defines: who decides, how, and when

### Code of Conduct
- Behavioral expectations for community members
- Enforced by moderation team

### PR (Pull Request)
- Proposed code change submitted for review
- GitHub-based workflow

### Issue
- Bug report, feature request, or task
- Tracked on GitHub Issues

### Good First Issue
- Beginner-friendly task labeled for newcomers
- Includes: clear description, suggested approach, mentor

### Hackathon
- Time-limited event focused on building
- Common themes: plugins, docs, bug fixes

### Contributor Summit
- In-person gathering of top contributors
- Annual event for planning and community building

### Discord
- Real-time chat platform for community
- API-OSS primary community hub

### GitHub Discussions
- Forum-style conversations on GitHub
- Used for: Q&A, ideas, show and tell

### Newsletter
- Weekly email digest of project updates
- Keeps community informed

### Sticker / Swag
- Physical merchandise for contributors
- Stickers, t-shirts, hoodies as thank-you gifts

### Community Edition
- Free, open-source version of API-OSS
- All core features included; no artificial limits
