# Glossary 23: Mobile Glossary

## Terms

### PWA (Progressive Web App)
- Web application that can be installed as a native app
- API-OSS provides PWA for mobile access

### Offline Mode
- Full functionality without internet connection
- API-OSS supports offline operation with local models

### Service Worker
- Browser script that enables offline functionality
- API-OSS PWA uses service workers for caching

### Cache API
- Browser storage for offline assets and data
- API-OSS caches: models, documents, search index

### IndexedDB
- Browser-based NoSQL database for offline data
- API-OSS stores user data for offline access

### Background Sync
- Syncs data when connectivity is restored
- API-OSS queues offline changes and syncs automatically

### Push Notification
- Server-to-client notification without app open
- API-OSS supports push via Service Worker API

### App Shell
- Minimal app UI loaded first for instant startup
- API-OSS PWA uses app shell architecture

### Manifest (Web App Manifest)
- JSON file defining PWA metadata
- Enables "Add to Home Screen" on mobile

### Mobile UI
- Touch-optimized interface for phones/tablets
- API-OSS responsive design adapts to all screen sizes

### QR Sharing
- Share API-OSS access via QR code
- Quick connect for mobile devices

### P2P Sync (Mobile)
- Direct device-to-device sync without server
- API-OSS P2P works over WiFi Direct or LAN

### Touch Gestures
- Touch-based interaction: swipe, pinch, tap
- API-OSS mobile UI supports common gestures

### Mobile Bridge
- Using API-OSS bridge on mobile platforms
- Interact via Telegram, WhatsApp, or other chat apps

### Data Usage (Mobile)
- Bandwidth consumption on cellular networks
- API-OSS optimizes for minimal data usage
