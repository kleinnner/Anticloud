# Glossary 12: Bridge Glossary

## Terms

### Bridge
- Connector that relays messages between API-OSS and external platforms
- Enables AI interaction through chat apps

### Discord Bridge
- Connects API-OSS to Discord servers
- Supports: threads, slash commands, file attachments, threads

### Telegram Bridge
- Connects API-OSS to Telegram
- Supports: groups, channels, inline mode, file sharing

### WhatsApp Bridge
- Connects API-OSS to WhatsApp
- Supports: individual chats, groups, media messages

### Slack Bridge
- Connects API-OSS to Slack workspaces
- Supports: channels, DMs, slash commands, message threads

### Matrix Bridge
- Connects API-OSS to Matrix (decentralized chat)
- Supports: rooms, direct messages, file sharing

### Signal Bridge
- Connects API-OSS to Signal messenger
- Supports: end-to-end encrypted messaging

### Email Bridge
- Converts email to/from API-OSS messages
- Supports: IMAP/POP3 receive, SMTP send

### SMS Bridge
- Connects API-OSS to SMS gateways
- Supports: Twilio, Vonage, custom gateways

### Bridge Configuration
- Settings for a bridge instance
- Includes: API key, platform target, channel mapping, permissions

### Channel Mapping
- Defines which API-OSS conversations map to which external channels
- Supports 1:1, 1:many, many:1 mappings

### Bridge Server
- Dedicated process running bridge connections
- Can run on same host or separate server

### Bridge Authentication
- How the bridge authenticates to the external platform
- Methods: API token, OAuth, bot token, username/password

### Rate Limit
- Maximum messages per time period the bridge can send
- Prevents abuse and platform throttling
