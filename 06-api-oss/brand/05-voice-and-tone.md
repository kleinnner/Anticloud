# Voice and Tone

## Overview

Brand voice and tone guidelines for API-OSS.

## Brand Voice

```yaml
characteristics:
  - Clear: No jargon without explanation
  - Direct: Get to the point
  - Technical: Respect the audience's knowledge
  - Friendly: Approachable, not corporate
  - Confident: We know our stuff
  - Humble: Open source community spirit
```

## Voice Examples

```yaml
good:
  - "Deploy API-OSS in 5 minutes."
  - "Your data never leaves your infrastructure."
  - "Switch between OpenAI, Anthropic, or any model."
  - "Because you shouldn't have to choose between AI and privacy."

avoid:
  - "Leverage synergistic paradigms..."
  - "We're disrupting the API space..."
  - "Best-in-class enterprise-grade solution..."
  - "Contact sales for pricing (hide it)"
```

## Tone by Context

```yaml
documentation:
  style: Technical, precise, complete
  example: "To configure rate limiting, add the following to your config.yml..."

blog:
  style: Conversational, informative
  example: "Here's how we built a gateway that handles 100K requests per second."

error_messages:
  style: Helpful, actionable
  example: "Invalid API key. Create a new key at https://app.api-oss.local/keys"

marketing:
  style: Benefit-focused, clear
  example: "Run any AI model on your own infrastructure."
```

## Words to Use

```yaml
prefer:
  - gateway (not proxy)
  - sovereign (not secure)
  - open source (not free)
  - deploy (not install)
  - configure (not setup)

avoid:
  - synergy, paradigm, leverage
  - disruptive, revolutionary
  - best-in-class, industry-leading
  - granular, holistic
```
