# Governance Overview

## Introduction

API-OSS governance model ensures transparent, community-driven development of the open-source project.

## Governance Structure

```
┌─────────────────────────────────────────────┐
│              Community Members               │
├─────────────────────────────────────────────┤
│              Contributors                    │
├─────────────────────────────────────────────┤
│              Maintainers                     │
├─────────────────────────────────────────────┤
│            Core Committers                   │
├─────────────────────────────────────────────┤
│   Technical Steering Committee (TSC)         │
├─────────────────────────────────────────────┤
│            Project Lead                      │
└─────────────────────────────────────────────┘
```

## Decision-Making

| Decision Type | Who Decides | Process |
|---|---|---|
| Technical direction | TSC | Consensus |
| Feature inclusion | TSC | RFC + vote |
| Release management | Core committers | Checklist |
| Community guidelines | Maintainers | Discussion |
| Code of conduct | TSC | Policy |
| Funding/spending | Project lead | + TSC approval |

## Communication

```
TSC meetings: Bi-weekly (public)
Community calls: Monthly (public)
RFC review: Asynchronous (GitHub)
Decision records: ADR format
```
