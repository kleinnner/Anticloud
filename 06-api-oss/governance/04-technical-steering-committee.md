# Technical Steering Committee (TSC)

## Overview

The TSC provides technical oversight and direction for API-OSS.

## Composition

```
- 5–7 members
- Mix of core committers and community representatives
- 1-year terms
- Staggered elections (2–3 seats per cycle)
```

## Responsibilities

```
- Approve/reject RFCs
- Set technical direction
- Manage release process
- Resolve technical disputes
- Appoint core committers
- Oversee security response
```

## Meetings

```
Schedule: Bi-weekly, alternating time zones
Duration: 60 minutes
Format: Public (recorded)
Agenda: GitHub issues with `tsc-agenda` label
```

## Decision Process

```
- Consensus-based
- If no consensus: majority vote
- Project lead has veto (with explanation)
- Tie-breaking: rotating chair
```

## Becoming a Member

```
1. Core committer for 6+ months
2. Nominated by existing member
3. Community vote (7 days)
4. Appointed by TSC
```
