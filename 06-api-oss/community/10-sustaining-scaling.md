# Community Playbook 10: Sustaining & Scaling

## Overview

How to sustain community momentum and scale as the project grows.

## Phase-Based Strategy

### Phase 1: Bootstrap (0–1,000 members)

```yaml
Focus: Build core, establish culture
Team: Founder + 1–2 community members
Activities:
  - Daily personal engagement (answer every question)
  - Weekly AMAs with founder
  - Manual onboarding of every new member
  - Find first 10 super-contributors

Key metric: Active members (DAU > 50)
Budget: $500/month
```

### Phase 2: Growth (1,000–10,000 members)

```yaml
Focus: Systematize, delegate
Team: 1 dedicated community manager + 5 volunteer mods
Activities:
  - Establish community guidelines
  - Create moderation team
  - Automate onboarding
  - Start events program

Key metric: Weekly active contributors > 20
Budget: $3,000/month
```

### Phase 3: Scale (10,000–100,000 members)

```yaml
Focus: Autonomous, self-sustaining
Team: 2–3 community managers + 20 volunteer mods
Activities:
  - Local chapters (global)
  - Regional events
  - Contributor program with stipends
  - Plugin marketplace (community-run)

Key metric: Community-sourced PRs > 50% of all PRs
Budget: $15,000/month
```

## Scaling Challenges

| Challenge | Solution |
|-----------|----------|
| Signal/noise ratio | Multiple channels, threads, auto-mod |
| Mod burnout | Rotate mods, limit shifts, paid mods for scale |
| Cliques forming | Mixer events, rotation of discussion topics |
| Newcomer overwhelmed | Better onboarding, buddy system |
| Language barriers | Translation bots, language-specific channels |
| Time zones | Rotate community manager hours, regional mods |

## Sustainability Principles

```yaml
1. Community-owned, not vendor-owned
   - Community should survive without I or any single person
   - Document everything
   - Share decision-making

2. Self-serve over hand-holding
   - Automated onboarding
   - Self-serve support (docs, FAQ, AI bot)
   - Community-powered support

3. Recognition > compensation
   - Most contributors want impact, not money
   - Recognition scales, compensation doesn't
   - But: stipends for critical contributors

4. Low ceremony, high action
   - Minimal process for contributions
   - Ship fast, iterate
   - Don't require meetings for decisions
```

## Burnout Prevention

```yaml
For community managers:
  - 4-day work week (sustainable pace)
  - Rotate on-call responsibility
  - Regular time off (disconnect)
  - Professional development budget

For volunteer moderators:
  - Max 2-hour shift per day
  - Monthly "mod appreciation" gift
  - Clear boundaries (when to escalate)
  - Co-mod system (never alone)

For contributors:
  - No pressure to contribute
  - Respond to "I'm burnt out" with grace
  - Temporary role reduction without stigma
```

## Long-Term Vision

```yaml
Year 5 goals:
  - 100,000+ community members
  - 10,000+ active contributors
  - 1,000+ plugins in marketplace
  - Community revenue > $1M/year
  - Community-elected governance board
  - API-OSS Foundation (independent nonprofit)
```

## Final Thoughts

Building a community is like gardening:
- Plant seeds (initial members)
- Water regularly (engagement)
- Pull weeds (moderation)
- Give sunlight (recognition)
- And be patient — it takes years to grow a garden.

The best open-source communities feel like a group of friends building something amazing together. That's the goal.
