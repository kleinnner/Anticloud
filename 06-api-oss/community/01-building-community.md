# Community Playbook 1: Building a Developer Community

## Overview

How to build an active, engaged developer community around API-OSS.

## Why Community Matters

| Benefit | Impact |
|---------|--------|
| Plugin ecosystem | 10x more plugins than you can build alone |
| Documentation | Community fixes gaps, adds examples |
| Bug reports | Early detection of edge cases |
| Word of mouth | Most effective marketing channel |
| Talent pipeline | Community members become contributors → employees |
| Support | Community helps each other (reduces your support load) |

## Community Platform Strategy

| Platform | Purpose | Priority |
|----------|---------|----------|
| Discord | Real-time chat, support, community | Primary |
| GitHub | Code, issues, PRs, discussions | Primary |
| Documentation | Official docs + community wiki | Secondary |
| Forum (Discourse) | Long-form discussions, RFCs | Optional |
| Twitter/LinkedIn | Announcements, thought leadership | Secondary |
| YouTube | Tutorials, demos | Optional |

## Launch Plan

### Month 1: Foundation

```yaml
Week 1-2:
  - Set up Discord server (channels: #general, #support, #showcase, #plugins)
  - Create GitHub Discussions (announcements, ideas, Q&A)
  - Write Code of Conduct
  - Invite first 50 users from existing contacts

Week 3-4:
  - Publish "Contributing Guide"
  - Create Good First Issue labels
  - Host first Discord AMA (ask me anything)
  - Aim: 100 members
```

### Month 2: Growth

```yaml
Week 5-6:
  - Launch community plugin showcase (monthly winners)
  - Write 3 "Contributor Spotlight" posts
  - Start weekly "Community Standup" (Discord voice)

Week 7-8:
  - Launch mentorship program for new contributors
  - Create contribution leaderboard
  - Host first community hackathon (virtual)
  - Aim: 500 members
```

### Month 3+: Sustained

```yaml
Ongoing:
  - Monthly community newsletter
  - Weekly standups (rotate hosts)
  - Quarterly hackathons
  - Annual contributor summit (in-person)
  - Maintain 10:1 member-to-contributor pipeline
```

## Community Health Metrics

| Metric | Target | How to Improve |
|--------|--------|----------------|
| Active members (DAU/MAU) | >20% MAU | Events, challenges, engagement |
| Messages/week | >100 in #general | Moderation, chat prompts |
| PRs from community | >30% of total | Good first issues, mentorship |
| First reply time | <5 minutes | Mod team, busy hours coverage |
| Member growth | 10–20% MoM | Marketing, events, word of mouth |
| Churn | <10% monthly | Onboarding, value delivery |

## Common Pitfalls

1. **Ghost town** — Community without activity → deletes itself
2. **Toxic behavior** — Failing to moderate → drives away good contributors
3. **No clear path** — People want to help but don't know how
4. **Over-moderation** — Too many rules stifles conversation
5. **No appreciation** — Contributors leave if work isn't recognized
