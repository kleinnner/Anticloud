# Community Playbook 3: Community Events & Hackathons

## Overview

Run events that drive engagement, contribution, and community growth.

## Event Types

### Virtual Hackathons

```yaml
Frequency: Quarterly
Duration: 1–2 weeks
Format: Online, async
Prizes: $500–$2,000 + swag

Themes:
  - Plugin marketplace (build a WASM plugin)
  - Documentation (improve docs, add examples)
  - Integration (new connector/bridge)
  - Bug bash (fix the most bugs)

Execution:
  - Kickoff: Discord live stream + issue list
  - Mid-point: Progress check-in + office hours
  - Judging: Maintainers + community vote
  - Awards: Announcement + blog post
```

### Contributor Summits

```yaml
Frequency: Annual
Duration: 2–3 days
Format: In-person (rotating location)
Cost: $100–$500 per attendee (subsidized)

Activities:
  - Day 1: Talks + workshops
  - Day 2: Hackathon + planning sessions
  - Day 3: Outdoor activity + social

Goals:
  - Build relationships (key for distributed teams)
  - Strategic planning for next year
  - Cross-pollinate ideas
  - Fun (retention)
```

### Meetups

```yaml
Frequency: Monthly (local chapters)
Format: In-person (hosted by community members)

Agenda:
  - 30min: Social + food/drinks
  - 30min: Talk (someone presents their API-OSS project)
  - 30min: Open discussion / Q&A
  - 30min: Social

Support:
  - Provide speaker list + slide templates
  - Cover food ($150–$300 per meetup)
  - Promote on social media
  - Organizer kit (stickers, banners)
```

### AMAs (Ask Me Anything)

```yaml
Frequency: Bi-weekly
Format: Discord text channel (async)

Process:
  - Announce topic + guest 1 week ahead
  - Collect questions (upvote system)
  - Guest answers over 2 hours
  - Transcript posted on blog

Guests:
  - Core maintainers
  - Plugin authors
  - Enterprise users
  - Industry experts
```

## Event Checklist

```yaml
Pre-event (4 weeks):
  [ ] Set date and theme
  [ ] Recruit judges/mentors
  [ ] Create issue list (for hackathons)
  [ ] Design promotional graphics
  [ ] Set up event page (lu.ma, eventbrite)

Pre-event (1 week):
  [ ] Promote on Discord, Twitter, newsletter
  [ ] Prepare prize distribution plan
  [ ] Test streaming setup (if live)

Day of event:
  [ ] Welcome message + instructions
  [ ] Active moderation/mentoring
  [ ] Track participation

Post-event:
  [ ] Thank participants
  [ ] Send prizes (within 1 week)
  [ ] Blog post: results + highlights
  [ ] Survey: what to improve
  [ ] Update community stats
```

## Measuring Success

| Metric | Target | How to Track |
|--------|--------|-------------|
| Attendance | 100+ (virtual), 30+ (in-person) | Registration count |
| PRs submitted | >20 (hackathon) | GitHub count |
| Retention | >50% return for next event | Survey |
| NPS | >50 | Post-event survey |
| New contributors | >10 per event | GitHub first-time |
