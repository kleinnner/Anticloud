# Brand Assets & Guidelines

## Overview

Brand identity guidelines for API-OSS.

## Logo

### Primary Logo

```text
Horizontal lockup with icon + "API-OSS" wordmark
Preferred for most use cases
Minimum width: 200px
Clear space: 50% of logo height on all sides
```

### Icon

```text
Square icon for avatars, favicons, app icons
Recommended size: 512×512px
```

### Variations

```
- Full color (primary)
- White (dark backgrounds)
- Black (light backgrounds, print)
```

## Color Palette

### Primary

```yaml
indigo:
  hex: "#6366F1"
  rgb: "99, 102, 241"
  usage: Primary brand color, buttons, links

emerald:
  hex: "#10B981"
  rgb: "16, 185, 129"
  usage: Success states, positive metrics
```

### Neutral

```yaml
slate-900:
  hex: "#0F172A"
  usage: Background (dark mode), headings

slate-50:
  hex: "#F8FAFC"
  usage: Background (light mode)
```

### Semantic

```yaml
red:
  hex: "#EF4444"
  usage: Errors, alerts, critical states

amber:
  hex: "#F59E0B"
  usage: Warnings, medium alerts

blue:
  hex: "#3B82F6"
  usage: Information, links
```

## Typography

```yaml
Headings: Inter (Bold, 700)
Body: Inter (Regular, 400)
Code: JetBrains Mono (Regular, 400)
```

## Logo Usage

```
- Do not stretch or distort
- Do not change colors
- Do not add effects (shadows, gradients)
- Do not rotate
- Maintain clear space
- Prefer transparent backgrounds
```

## Brand Voice

```
- Clear and direct
- Technical but accessible
- Confident but not arrogant
- Open source community tone
```
