# AI FAQ

## How does the AI work?

Liber (the Libern AI assistant) uses a pluggable engine architecture. The AI system is implemented in `crates/libern-core/src/ai/` with the following components:

1. **AiEngine trait** (`engine.rs`): Defines the interface all engines must implement.
2. **MockEngine** (`engine.rs`): Returns canned responses for development/testing.
3. **CandleEngine** (`qwen_engine.rs`): Runs the Qwen 2.5 1.5B model via llama.cpp CLI.
4. **Pipeline** (`pipeline.rs`): Builds chat prompts and packs context into token windows.
5. **RAG** (`rag.rs`): Retrieval-Augmented Generation for querying uploaded documents.
6. **Moderator** (`moderator.rs`): Content moderation using keyword filters + AI.
7. **Summarizer** (`summarizer.rs`): Channel conversation summarization.
8. **Reward model** (`reward.rs`): RLHF feedback system for preference learning.

```rust
pub trait AiEngine: Send + 'static {
    fn infer(&mut self, request: InferenceRequest) -> Result<(), String>;
    fn embed(&mut self, text: &str) -> Result<Vec<f32>, String>;
    fn is_loaded(&self) -> bool;
    fn model_info(&self) -> ModelInfo;
}
```

## Is the AI private?

**Yes.** The AI runs entirely on your machine. No data is sent to any external server. Everything — including the model file, inference execution, and conversation history — stays on your local device.

Key privacy guarantees:
- Zero telemetry: No usage data is collected.
- No cloud dependency: No API calls to OpenAI, Anthropic, or any external AI service.
- Local model execution: The GGUF model runs via llama.cpp on your CPU.
- Local conversation history: Stored in your local SQLite database.

## What model does Liber use?

Liber uses **Qwen 2.5 1.5B Instruct** in GGUF format, quantized to **Q4_K_M**.

- **Model**: Qwen/Qwen2-VL-2B-Instruct-GGUF
- **Quantization**: Q4_K_M (4-bit, ~1.1 GB)
- **Context size**: 4096 tokens
- **Source**: https://huggingface.co/Qwen/Qwen2-VL-2B-Instruct-GGUF

The model is downloaded from Hugging Face and cached locally. The download URL:
```
https://huggingface.co/Qwen/Qwen2-VL-2B-Instruct-GGUF/resolve/main/Qwen2-VL-2B-Instruct-Q4_K_M.gguf
```

## Can I use my own model?

Yes. The `CandleEngine` (`qwen_engine.rs`) runs any GGUF model compatible with llama.cpp. To use your own model:

1. Download your GGUF model file.
2. Place it in `{app_data_dir}/models/`.
3. Ensure it is named `Qwen2-VL-2B-Instruct-Q4_K_M.gguf` (or modify the path in `lib.rs`).
4. Place the `llama-cli` binary in `{app_data_dir}/bin/` or the app's `binaries/` directory.

The engine selection logic in `lib.rs`:
```rust
if model_path.exists() && binary_path.exists() {
    // Use CandleEngine with real model
    Box::new(CandleEngine::new(model_path, binary_dir))
} else {
    // Fall back to MockEngine
    Box::new(MockEngine::new())
}
```

## Do I need a GPU?

No. Liber's AI runs on CPU. The Qwen 2.5 1.5B model at Q4_K_M quantization runs reasonably well on modern CPUs. However, for faster inference:

- **AVX2 support** on your CPU significantly improves performance.
- **GPU acceleration** is not currently supported (llama.cpp can use CUDA, but the current CandleEngine uses CPU-only mode).
- **Apple Silicon** (M1/M2/M3) provides good performance due to unified memory.

The health diagnostics include a GPU check:
```
gpu_available: warn — "No GPU detected, using CPU"
```

## What if I don't download the model?

If you skip the model download, Liber uses `MockEngine` which returns canned responses. You will see responses like:

The `MockEngine` also generates mock embeddings for RAG. It's useful for testing and development without needing the full 1.1 GB model download.

## How does the AI context window work?

The `pack_context` function (`pipeline.rs`) packs recent conversation history into a context window:
```rust
pub fn pack_context(messages: &[(String, String)], max_tokens: usize, approx_chars_per_token: usize) -> Vec<String> {
    let mut packed = Vec::new();
    let mut total_chars = 0;
    // Start from most recent, add messages until we hit the token limit
    for (role, content) in messages.iter().rev() {
        let msg_len = role.len() + content.len() + 10; // overhead
        if total_chars + msg_len > max_tokens * approx_chars_per_token {
            break;
        }
        packed.push(format!("{}: {}", role, content));
        total_chars += msg_len;
    }
    packed.reverse();
    packed
}
```

The `ask_libern` command uses the last 20 messages from the `ai_conversations` table as context. The default max is 2048 tokens with ~4 characters per token approximation.

## How does the RAG system work?

The RAG (Retrieval-Augmented Generation) system in `rag.rs`:

1. **Document ingestion**: Documents are split into chunks (default 128 words), each chunk is embedded into a 128-dimensional vector.
2. **Storage**: Embeddings are stored as raw f32 bytes in the `document_chunks` table.
3. **Query**: User's question is embedded, cosine similarity is computed against all document chunks.
4. **Context**: Top 5 most relevant chunks are retrieved and injected into the prompt.

```rust
fn cosine_similarity(a: &[f32], b: &[f32]) -> f32 {
    let dot: f32 = a.iter().zip(b.iter()).map(|(x, y)| x * y).sum();
    let mag_a: f32 = a.iter().map(|x| x * x).sum::<f32>().sqrt();
    let mag_b: f32 = b.iter().map(|x| x * x).sum::<f32>().sqrt();
    if mag_a == 0.0 || mag_b == 0.0 { return 0.0; }
    dot / (mag_a * mag_b)
}
```

## How does the content moderation work?

The moderator in `moderator.rs` uses a two-stage approach:

1. **Keyword filter**: Fast pre-filter against blocked/flagged pattern lists.
2. **AI classification**: If the keyword filter doesn't trigger, the AI engine classifies the message with a prompt:
   > "Classify the following message as SAFE, FLAG, or BLOCK."

Results are returned as:
```json
{
  "classification": "Safe" | "Flag" | "Block",
  "reason": "...",
  "confidence": 0.95
}
```

### Moderation Categories

| Classification | Meaning | Action |
|---------------|---------|--------|
| Safe | No issues | Message allowed |
| Flag | Potentially problematic | Warning added, message visible |
| Block | Clearly violating | Message hidden, reported |

## How does the reward model (RLHF) work?

The reward system in `reward.rs` learns user preferences from feedback:

- Users rate AI responses as upvote (+1), neutral (0), or downvote (-1).
- **Preference profiles** track:
  - `temperature_bias`: Adjusted based on average score (-0.3 to +0.3)
  - `style_prompt`: Built from top-rated categories (e.g., "concise", "detailed")
  - `top_categories`: Most upvoted style categories
- The preference profile is used to adjust future responses:
  ```rust
  // Lower temperature for negative feedback (more deterministic)
  // Higher temperature for positive feedback (more creative)
  profile.temperature_bias = (profile.avg_score * 0.1).clamp(-0.3, 0.3);
  ```

## Troubleshooting AI

| Issue | Likely Cause | Solution |
|-------|-------------|----------|
| AI not responding | Model not downloaded | Download model via Settings > AI |
| Generic responses | MockEngine active | Download real model file |
| Slow responses | CPU-only inference | Close other apps, use GPU if available |
| Out of memory | Model too large for RAM | Use smaller model |
| "Engine not initialized" | Model file corrupted | Re-download model |
| Garbled output | Model version mismatch | Update model, restart |

## AI FAQ Extended

### Can I run multiple AI requests simultaneously?

By default, Libern limits to 1 concurrent AI request. You can increase this:
```json
{ "ai": { "max_concurrent": 2 } }
```
But beware of memory constraints — each concurrent request loads the model state.

### Does the AI support streaming?

Yes. All AI responses are streamed token-by-token via Tauri IPC Channel. The UI shows:
- Animated typing indicator during streaming
- Blinking cursor at end of text
- Full response when complete
- Rating buttons (thumbs up/down) for RLHF

### Can I use a different language model?

Yes. Download any GGUF-compatible model and place it in the models directory. The engine will try to load it. Note that system prompts are optimized for Qwen 2.5 Instruct.

### How much RAM does the AI need?

- Without model: ~200-400 MB
- With Qwen 2.5 1.5B (Q4_K_M): ~2-3 GB additional during inference
- Peak usage during inference: ~3.5 GB total

### Does Liber support batch processing?

The `batch_size` setting controls how many tokens are processed at once. Higher values use more RAM but may improve throughput. Default is 4.

### How do I reset the AI engine?

```typescript
// Force engine reload
import { invoke } from "@tauri-apps/api/core";
await invoke("reset_ai_engine");
```
