# Software Rasterizer ‚ÄĒ CPU Tile-Based Renderer

Kazkade's software rasterizer is a SIMD-accelerated, tile-based CPU renderer targeting the installer's live 3D cube visualisation and diagnostic overlays. It operates entirely on the CPU with no GPU dependency, making it suitable for headless servers, early boot, and constrained environments.

## Pipeline Stages

```mermaid
flowchart LR
    A[Vertex Fetch] --> B[Vertex Shader<br/>SIMD transform]
    B --> C[Tiler<br/>Bin triangles to tiles]
    C --> D[Tile Rasterizer<br/>Barycentric, coverage]
    D --> E[Fragment Shader<br/>SIMD interpolate + shade]
    E --> F[Output Merger<br/>Depth test, alpha blend]
    F --> G[Framebuffer<br/>Double-buffered RGBA]
```

### 1. Vertex Fetch & Shader

Vertices are provided as an interleaved array of `[x, y, z, w]` packed in AoS layout. The vertex shader applies a 4√ó4 model-view-projection matrix using SIMD dot products:

```rust

    // AVX2: 8 vertices per iteration via 8√ó4 matrix-vector
    // NEON: 4 vertices via FMLA
}
```

### 2. Tiler (Bin to Tiles)

The screen is divided into 32√ó32 pixel tiles. Each triangle is clipped to the viewport, then the tiler computes which tiles it touches using a bounding-box scan. Tile lists are stored per-tile in a flat array of triangle indices, avoiding dynamic allocation per frame.

### 3. Tile Rasterizer

Each tile is processed independently ‚ÄĒ trivially parallelisable with a thread pool. For every triangle in the tile's list:

- **Barycentric coordinates** are computed for each pixel using edge functions.
- **Early‚ÄĎZ culling** discards occluded fragments before shading.
- **Coverage masks** are computed in 8√ó8 sub-blocks using SIMD compare instructions.

The rasterizer emits fragments as a struct-of-arrays: `{x, y, z, w, u, v, coverage}`.

### 4. Fragment Shader

Fragment shading applies interpolated vertex attributes (colour, normals, texture coordinates) using SIMD `vfmadd` for perspective-correct interpolation. Simple Blinn-Phong lighting is computed in the shader with no branching ‚ÄĒ all control flow is predicated via blend masks.

### 5. Output Merger

The output merger performs depth testing (`<` by default) and alpha blending on the 32‚ÄĎbpp RGBA framebuffer. The depth buffer is stored as `f32` for precision.

## Double-Buffered Framebuffer

The framebuffer is a pair of `Vec<u32>` of size `width √ó height`. At any time, one buffer is being rendered into and the other is being displayed (via `winit` raw window handle or file dump). Swap is a pointer swap ‚ÄĒ no copy.

```rust
pub struct Framebuffer {
    front: Vec<u32>,
    back: Vec<u32>,
    width: u32,
    height: u32,
}

impl Framebuffer {
    pub fn swap(&mut self) {
        std::mem::swap(&mut self.front, &mut self.back);
    }
}
```

## Parallelism

Tile rasterisation is dispatched to a `rayon` thread pool. Each thread processes one or more tiles independently, writing only to its assigned tile region. The output merger in a separate pass resolves the back buffer ‚ÄĒ no locks are needed because tiles do not overlap.

## Performance

On a Ryzen 9 7950X (16 cores), the rasterizer sustains 60 FPS at 1920√ó1080 for a 10K‚ÄĎtriangle scene with Blinn-Phong shading. The tile-based design keeps each thread's working set in L2 cache.
