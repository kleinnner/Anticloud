## Introduction

MF+SO has a vibrant, helpful community of users ranging from complete beginners to security professionals. Whether you're stuck on a setup step, curious about a feature, or want to contribute, there's a place for you.

### Support Channels Overview

| Channel | Best For | Response Time | Availability |
|---------|----------|---------------|--------------|
| Community Forum | Detailed questions, guides, discussions | Usually within hours | 24/7 |
| Discord | Quick questions, real-time chat | Usually within minutes | 24/7 |
| GitHub | Bug reports, feature requests, development | Within 1-2 days | 24/7 |
| Email Support | Account issues, security concerns | Within 1 business day | Business hours |
| In-App Help | Quick answers, guided tutorials | Instant | 24/7 |

### Before Asking for Help

To get the fastest help:
1. **Search the Knowledge Base** at https://docs.mfso.app
2. **Search the Forum** — your question may already be answered
3. **Check the FAQ** in the in-app help center
4. **Try restarting** the app (solves many issues)
5. **Check for updates** — your issue may be fixed in the latest version

## Community Forum

### Accessing the Forum

- **Web**: https://community.mfso.app
- **In-app**: Help → Community Forum

### Forum Categories

| Category | Description |
|----------|-------------|
| **Getting Started** | Setup, installation, first vault |
| **Using MF+SO** | Features, tips, workflows |
| **Security & Privacy** | Encryption, best practices, threats |
| **Backup & Recovery** | Seed phrases, exports, recovery |
| **Troubleshooting** | Bugs, issues, workarounds |
| **Feature Requests** | Suggest and discuss new features |
| **Tips & Tricks** | Power user tips, hidden features |
| **Announcements** | Release notes, important updates |
| **Off-Topic** | Non-MF+SO discussions (community lounge) |

### Creating a Post

1. Click "New Post"
2. Choose a category
3. Write a descriptive title:
   - Good: "Can't scan QR code on Android 14"
   - Bad: "Help please!"
4. Describe your issue:
   - What you were doing
   - What happened
   - What you expected
   - Steps to reproduce
5. Include relevant details:
   - App version (Settings → About)
   - Device type and OS version
   - Steps you've already tried
6. Click "Submit"

### Forum Etiquette

- **Search before posting**: Your question may already be answered
- **Be specific**: "My TOTP codes aren't working" is better than "It doesn't work"
- **Be patient**: Community members help in their free time
- **Be respectful**: Everyone was a beginner once
- **Mark solutions**: When your question is answered, mark the solution
- **Keep it public**: Help others by keeping conversations in the forum

### Forum Badges

Community members earn badges for participation:
- **Helper**: Answered 10 questions
- **Expert**: Answered 50 questions with solutions
- **Moderator**: Trusted community member who helps moderate
- **Contributor**: Created helpful guides or translations
- **Early Adopter**: Joined in the first 3 months

## GitHub

### Repository Access

- **Main repo**: https://github.com/mfso/mfso
- **Documentation**: https://github.com/mfso/docs
- **Desktop app**: https://github.com/mfso/desktop
- **Mobile app**: https://github.com/mfso/mobile
- **CLI**: https://github.com/mfso/cli

### Reporting Bugs

1. Go to the appropriate repository
2. Click "Issues" → "New Issue"
3. Choose the bug report template
4. Fill in:
   - **Description**: What's wrong
   - **Steps to Reproduce**: Exact steps
   - **Expected vs Actual Behavior**: What you expected vs what happened
   - **Screenshots/Videos**: Visual evidence (if applicable)
   - **Environment**: OS, app version, device
   - **Logs**: Relevant error logs (if available)
5. Label appropriately (bug, security, performance)

### Feature Requests

1. Go to the main repository
2. Click "Issues" → "New Issue"
3. Choose the feature request template
4. Describe:
   - **Problem**: What problem would this solve?
   - **Solution**: What would you like to see?
   - **Alternatives**: What have you tried?
   - **Priority**: How important is this to you?

### Contributing Code

See the CONTRIBUTING.md file in each repository. General guidelines:
1. Fork the repository
2. Create a feature branch
3. Write tests for your changes
4. Ensure all tests pass
5. Submit a pull request
6. Respond to code review feedback

### Security Disclosures

**Do not** report security vulnerabilities publicly on GitHub.

Instead:
- Email: security@mfso.app
- Use our PGP key (available at https://mfso.app/security/pgp)
- Expect acknowledgment within 24 hours
- We follow responsible disclosure practices

## Email Support

### When to Use Email

Email support is for:
- **Account recovery issues** (can't access your vault)
- **Security concerns** (suspected compromise, phishing)
- **Billing issues** (subscription, payment)
- **Account deletion requests**
- **Enterprise support** (if you don't have a dedicated contact)
- **Bug bounty submissions**
- **Legal requests** (DMCA, GDPR, subpoenas)

### Email Addresses

| Purpose | Email | Response Time |
|---------|-------|---------------|
| General support | support@mfso.app | 1 business day |
| Security issues | security@mfso.app | 24 hours |
| Privacy concerns | privacy@mfso.app | 1 business day |
| Enterprise | enterprise@mfso.app | 4 business hours |
| Press inquiries | press@mfso.app | 1 business day |
| Legal | legal@mfso.app | 2 business days |

### Writing a Support Email

1. **Use a descriptive subject line**:
   - Good: "Account Recovery: Lost seed phrase, need help"
   - Bad: "Help"

2. **Include relevant details**:
   - Your account email
   - App version
   - Device type and OS
   - When the issue started
   - Steps you've already tried

3. **Attach relevant files** (if applicable):
   - Screenshots (no sensitive data)
   - Error logs
   - Export files (if relevant)

4. **Expect a response within 1 business day**

### What Email Support Cannot Do

- **We cannot reset your master password** (zero-knowledge architecture)
- **We cannot generate a new seed phrase** for you
- **We cannot access your vault contents**
- **We cannot decrypt your exports**

## In-App Help

### Help Center

Access the in-app help center:
- Desktop: Help → Help Center
- Mobile: Profile/Settings → Help

The help center includes:
- **Interactive tutorials**: Step-by-step guides within the app
- **Video guides**: Short videos showing common tasks
- **FAQ**: Frequently asked questions with answers
- **Troubleshooting**: Common issues and solutions
- **Glossary**: Terms and definitions

### Guided Tutorials

New features include guided tutorials:
1. When a new feature is added, a tutorial prompt appears
2. Tap "Start Tutorial" for an interactive walkthrough
3. The tutorial highlights relevant UI elements
4. Complete the tutorial to dismiss it

### Contextual Help

Tap the "?" icon anywhere in the app:
- Shows help relevant to the current screen
- Links to the full documentation
- Shows keyboard shortcuts (desktop)

### Diagnostic Tool

The in-app diagnostic tool helps with troubleshooting:
1. Help → Run Diagnostics
2. The tool checks:
   - Network connectivity
   - Server connectivity
   - Sync status
   - Time synchronization
   - App version
   - Vault integrity
3. Results show any issues found
4. You can export results to share with support

## Support Etiquette

### When Asking for Help

**Do**:
- Be clear and specific about your issue
- Include relevant details (version, device, OS)
- Search before asking
- Be patient — most help is volunteer-based
- Thank people who help you
- Mark solutions in forums

**Don't**:
- Demand immediate help
- Ping or DM individual helpers for support
- Post the same question in multiple channels
- Share sensitive information publicly
- Argue with helpers

### When Helping Others

**Do**:
- Be welcoming to beginners
- Provide clear, step-by-step guidance
- Link to documentation when relevant
- Ask clarifying questions
- Be patient with repeated questions

**Don't**:
- Be condescending or dismissive
- Give security advice you're not sure about
- Share code or configurations without explaining them
- Encourage unsafe practices

### Escalation

If a community member is being abusive:
1. Don't engage
2. Report to a moderator (use the report function)
3. Moderators will review and take appropriate action
